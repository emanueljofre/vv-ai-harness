Notable changes to the VisualVault Node.js server and client library (`lib/VVRestApi/`).

**Scope:** Server/library code changes and project-wide development tooling. For investigation work, see [`research/`](research/). For platform documentation, see [`docs/`](docs/).

Format follows [Keep a Changelog](https://keepachangelog.com/). Entries stay under **Unreleased** until a version is published to npm.

## [Unreleased]

### Changed

- `saveFormAndReload` helper navigates explicitly to the saved-record URL returned by `saveFormOnly()` instead of calling `page.reload()`. Under V2 on vv5dev, Angular does NOT push the `DataID` into the browser URL on save — the URL stays at `?formid=...` (template mode) — so `page.reload()` re-loaded an empty template and silently dropped the just-saved value. The symptom initially looked like a platform regression (`FORM-BUG-V2-SAVE-RELOAD-EMPTY`, since withdrawn); root cause was this helper. Verified 2026-04-22 against 4 previously-failing cat-14/cat-16 save-reload tests. cat-4-reload tests were previously passing accidentally because their URL contained `&FieldN=...` URL params that the reload re-applied — with the fix, they now exercise real save-reload persistence.
- `regression-reporter.js` deduplicates `onTestEnd` events by `(tcId, project)`, keeping only the last attempt. Playwright fires `onTestEnd` once per retry; previously a test that failed once and passed on retry counted as both a failed and a passed entry in the JSON. Summary counts and artifact generators now see a truthful post-retry view.
- `dash-export.spec.js` `parseXmlExport` auto-detects the row element instead of hardcoding `<DateTest>`. The XML export names its row element after the form template with spaces stripped (`Date Test Harness` → `<DateTestHarness>` on vv5dev, `<DateTest>` on vvdemo). Detector picks the opening tag whose count matches `<Form_x0020_ID>` count, with a fallback that excludes outer containers (`<VisualVault>`) and schema definitions (`<xs:*>`). Removes 4 per-TZ failures without customer-specific logic.
- `cat-14-mask-impact.spec.js` `sfvGfvPairs` map extended to include `.V2` siblings (`'14-C-GFV.V2': '14-C-SFV.V2'`, `'14-D-GFV.V2': '14-D-SFV.V2'`). GFV tests look up their paired SFV by exact ID; without the V2 entries, `.V2` GFV tests found no pair, skipped the SFV setup, and captured an empty field.
- `cat-4-reload.spec.js` scope filter in the `fillinRelateReload` branch moved to run immediately after source-form load, BEFORE the source `setFieldValue` + assertions. Previously V1 entries on a V2 env hit source-side assertions against the wrong expected values before the scope check could skip them.
- `cat-6-current-date.spec.js` V2-aware local-date extraction — for entries with `scope: 'V2'`, reads the date portion of the raw ISO string directly (V2 "fake-Z" serialization encodes local wall-clock with a `Z` suffix, so re-applying a timezone shifts the date by one day near emulated midnights). V1 (Date object) entries continue to use `toLocaleDateString` with the IANA TZ. Eliminates IST-chromium flakiness near 18:30 UTC.
- `audit-kendo-version.spec.js` Phase 2/3/3b use a V2-aware skip guard (`test.skip(await getCodePath(page), ...)`) instead of `typeof kendo !== 'undefined'`. Kendo v2 also exposes a `kendo` global but with an incompatible API; the codepath check is the real V1/V2 discriminator. Converts 8 per-run failures to clean skips on V2 envs.
- `audit-bug1-tz-stripping.spec.js` Phase 2/4/5 tests use `test.skip(await getCodePath(page), ...)` instead of hard-asserting `expect(getCodePath).toBe(false)`. These phases document V1-only behavior (call-trace, flag-flip, cross-TZ init); on V2 envs they now skip cleanly instead of failing. Phase 4 line 419's post-flag-flip `toBe(true)` check is preserved.
- `cat-12-edge-cases.spec.js` removed `test.setTimeout(15000)` for edge-input entries. The 15s cap predated the scope filter; on V2 envs it fired BEFORE the scope filter could skip V1-only entries, causing timeouts where clean skips were expected. The V2 FORM-BUG-8 hang is already handled by the scope filter (V1 entries skip on V2) — if V2 siblings are added later they can wrap the SFV call in `Promise.race` rather than capping the whole test.
- Expected-value corrections in `testing/fixtures/test-data.js` for 18 V2 entries exposed by the `saveFormAndReload` helper fix + mid-sweep review-queue triage:
    - 10 cat-4 FAR `.V2` entries: `expectedSourceRaw` / `expectedSourceGfv` / `expectedApi` patched to V2 observed values (source raw = stringified Date under V2 full normalization; GFV returns raw under V2).
    - 4 cat-14/cat-16 save-reload `.V2` entries (14-C-save, 14-D-save, 16-A-controls, 16-C-controls): `expectedRaw` / `expectedApi` patched from empty-string (artifact of the broken helper) to real persisted values.
    - 2 cat-7 epoch `.V2` entries (7-A-epoch, 7-C-epoch): `expectedApi` corrected to the stringified epoch (V2 preserves the number through both raw and GFV).
    - 1 cat-16 Config D entry (16-D-controls.V2) + 1 cat-3 cross-TZ entry (3-C-IST-BRT.V2) patched with real V2 values; stale V1 bug tags (`Bug #1`, `Bug #4`, `FORM-BUG-5`) removed where the V2 behavior no longer exhibits them.
    - All 6 entries from the review queue (`v2-review-queue.md`) tagged with appropriate `FORM-BUG-V2-*` tags or untagged where V2 behavior matches intended semantics. Review queue marked CLOSED.
- `captureFieldValues` helper normalizes raw/api values — Date objects → ISO string, numbers/booleans → string. V2 preserves Date objects and epoch numbers where V1 converts to strings; tests now compare semantic equivalents, not types. Observed original type is preserved as `rawType`/`apiType` for V1-vs-V2 drift detection. Fixes ~22 cat-7 failures that tripped purely on type mismatch.
- `saveFormOnly` also returns the auto-generated instance name (`VV.Form.VV.FormPartition.FormName`) alongside `dataId`/`url`. Enables dashboard row matching for dynamically-created test records.
- `global-setup.js` saved-records policy — stamped with the current platform build fingerprint at creation; reuses cached records only while the fingerprint matches. On platform rollover, records are recreated so tests never compare against records saved under a different build. Two new `RECORD_DEFINITIONS` entries (`db6-dateonly`, `db6-datetime`) populate Date Test Harness records covering all 8 configs for `dash-cross-layer` DB-6 tests. `SAVED_RECORDS` normalized to `{ url, dataId?, instanceName? }` per key.
- `dash-cross-layer`, `dash-filter`, `dash-export`, `dash-sort`, and `doc-index-field-dates` specs rewired to read URL + xcid/xcdid + field names from `customerTemplates` / `FIELD_MAP` / `customerDocConfig` instead of hardcoded vvdemo values. Doc spec adds a top-level skip when the active env has no doc-test assets (e.g., vv5dev where `docapi` is disabled). Dash-cross-layer test titles now include fine-grained `db-6-{config}-{field}` IDs for reporter indexing.
- `cat-6-current-date.spec.js` uses a semantic today-in-TZ comparator — parses raw as a Date regardless of V1 (Date object) or V2 (ISO string) and compares the calendar date in the entry's TZ. `rawType`/`apiType` annotations record the observed type for drift detection. Eliminates 14 V1-only `rawIsDate` assertion failures without weakening the real assertion.
- `regression-reporter.js` `_extractTcId()` extended to recognize bare-digit slot IDs (`15-vv-core`, `12-dst-brazil`) with em-dash/en-dash/colon separators, plus `DB-*`/`DOC-*`/`WF-*`/`SP-*`/`WS-*` category prefixes. Cat-15 audits and dashboard/doc tests now surface as indexable TC IDs instead of collapsing into unmatched audit observations.
- `CUSTOMER_DOC_CONFIG` keyed by `.env.json` customer key (`EmanuelJofre-vvdemo`, `EmanuelJofre-vv5dev`, `WADNR`) with `null` values for environments where `docapi` is disabled. Resolver falls back to `customerAlias` for backward-compat.
- Build-context capture is always fresh — removed the 1-hour cache in `testing/global-setup.js` `captureBuild()`. Stale build metadata defeats the fingerprint's purpose (correlating runs with rollouts); the ~2s HTTP cost is trivial on a multi-minute regression run. Auth-state cache is unaffected (logins remain expensive and are still cached). All outcome-producing tools (forms regression, WS regression, dash regression, extract manifests, `env:profile`) now record a build fingerprint captured at run time with no staleness window.

- **Bug reporting standards** — expanded the support-format template and shared conventions. `bug-reporting.md` adds principle #6 (frame around user-observable effects, not internal mechanism). `bug-report-support.md` replaces the old single-step repro block with a structured QA-style layout: Preconditions, Test data, Reproduction steps (single path OR parallel peer-level Reproduction A/B/C subsections with shared preconditions), and Other triggers reserved for genuinely secondary paths. New "Filling It Out Well" guidance added: frame around observable effects (with good/bad example), equal-hierarchy reproductions, atomic steps, on-screen value formats over API serialization formats.

- **AI harness architecture**: Restructured repo as an AI workbench with independent nested repos. `tasks/` renamed to `research/`. `scripts/server-scripts/` and `scripts/form-scripts/` reorganized into `scripts/examples/`. `lib/` converted to independent git repo (gitignored by harness). `README.md` and `Readme-microservices.md` moved to `lib/`. Project workspace artifacts (`extracts/`, `testing/`, `environment.json`) now gitignored. `.husky/pre-commit` simplified to lint-staged only (upstream guard removed). `.husky/upstream-allowlist`, `_config.yml`, `.npmignore` removed. Claude commands updated for multi-repo awareness. `[@]-create-project` supports `--repo` flag and non-VV projects.

- **Artifact routing restructure**: Execution artifacts (runs, summaries, results.md, status.md) moved from `tasks/` to `projects/{customer}/testing/{task}/{component}/`. Tasks now hold only platform methodology (matrix, test-cases, analysis). Matrix files stripped of results columns — per-env tracking in `status.md`. Generalized principle: "Is this artifact true regardless of customer/environment?" determines routing. All test commands, sharing rules, and CLAUDE.md files updated.
